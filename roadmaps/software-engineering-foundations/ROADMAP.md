# Software Engineering Foundations Roadmap

This roadmap builds the software engineering base first. AI Engineering work should wait until these foundations are strong.

## Topics

| Order | Topic | Main Evidence |
| ---- | ----- | ------------- |
| 01 | Programming Fundamentals | Notes, snippets, and small exercises |
| 02 | Computer Science Foundations | Notes, snippets, and written explanations |
| 03 | Data Structures and Algorithms | Implementations, tests, and problem notes |
| 04 | Git, GitHub, and CLI | Branch workflow, commits, and terminal practice |
| 05 | Web Fundamentals | HTML, CSS, JavaScript, HTTP notes |
| 06 | Frontend Engineering | Component-based UI exercises |
| 07 | Backend Engineering | Small API services |
| 08 | Databases | Schema design, queries, and migrations |
| 09 | API Design | REST contracts, validation, and docs |
| 10 | Auth and Security Basics | Auth flows, secrets handling, and threat notes |
| 11 | Testing and Debugging | Unit tests, integration tests, and debugging logs |
| 12 | DevOps and Deployment | Docker, deployment notes, and health checks |
| 13 | System Design | Architecture notes and ADRs |
| 14 | SWE Portfolio | Polished portfolio projects |

## Completion Standard

A topic is only done when its `mastery.md` contains concrete evidence. Reading alone is not enough.
